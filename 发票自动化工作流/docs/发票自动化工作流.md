# 发票自动化工作流

## 概述

从邮件/网盘收集发票 → PyMuPDF/Vision 提取信息 → 人工复核 → 按规则重命名归档 → Excel登记索引。

### 项目进度记录

当前进行中的项目进度记录在项目文件夹内的 `工作流状态.md` 中。
接手新会话时，先读这个文件了解：当前批次到哪一步了、哪些文件待确认、Excel同步状态、未决事项。
每次操作完必须更新这个文件，保持最新。

## 一、收集

从邮件附件、飞书云盘、网盘收集发票 PDF/JPG/PNG 等格式。
工具：IMAP / 飞书 Drive / browser。

## 二、解析

使用 PyMuPDF (fitz) 提取 PDF 文本内容。
扫描件和非PDF文件（jpg/png/jpeg）用 Vision 视觉识别。
已有完整文件名的PDF可直接从文件名正则提取信息，秒级处理。

### Agnes AI Vision API 调用方式（独立可运行）

> 不依赖 Hermes，任何 AI Agent 或开发者都能直接用。

**API 基本信息：**

| 项目 | 值 |
|------|-----|
| Base URL | `https://apihub.agnes-ai.com/v1` |
| 模型 | `agnes-2.0-flash` |
| 认证方式 | Bearer Token（API Key） |
| 兼容格式 | OpenAI Chat Completions API |

**调用方式：** 用 `/chat/completions` 端点，消息里同时传 text + image_url（base64 或公网 URL）。

**Python 调用示例：**

```python
import requests
import base64

API_KEY="你的API_KEY"
API_URL = "https://apihub.agnes-ai.com/v1/chat/completions"

# 读取图片转 base64
with open("发票.jpg", "rb") as f:
    img_b64 = base64.b64encode(f.read()).decode()

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

payload = {
    "model": "agnes-2.0-flash",
    "messages": [{
        "role": "user",
        "content": [
            {"type": "text", "text": "请识别这张发票图片中的收款方、付款方、金额、日期、费用类型，用JSON格式返回。"},
            {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{img_b64}"}}
        ]
    }],
    "max_tokens": 500,
    "temperature": 0.1
}

response = requests.post(API_URL, headers=headers, json=payload)
result = response.json()
content = result["choices"][0]["message"]["content"]
print(content)
```

**PDF 转图片再识别：**

```python
import fitz  # PyMuPDF

doc = fitz.open("发票.pdf")
page = doc[0]
mat = fitz.Matrix(2.0, 2.0)  # 2倍清晰度
pix = page.get_pixmap(matrix=mat)
img_data = pix.tobytes("png")
doc.close()

# img_data 写入临时文件后用上面的方式调 API
```

**批量处理要点：**
- `temperature=0.1`（越低越稳定）
- `max_tokens=500`（够了）
- 每张之间加 `time.sleep(0.5~1.0)` 避免限流
- 每处理 5-10 张保存一次进度，防中断丢数据

**完整独立脚本位置：** `invoice_recognize.py`
**详细 API 说明位置：** `API调用说明.md`

## 三、识别与重命名

提取：收款方、付款方、金额、日期、费用类型。
按命名规则重命名：`YYYYMMDDHHMM-分类-收款方-付款方-$金额.扩展名`
日期时间为12位（如 202508210053）。
扩展名保持原文件类型（pdf/jpg/png/jpeg）。

## 四、分类与去向判断

### 分类规则

1. **律所发票** — 律所自开发票（如 WUKONG），不参与客户发票流程
2. **个人发票** — 个人消费性质的发票（currency exchange、cash withdraw 等）
3. **USPS跳过** — 无法处理或特殊标记的
4. **普通发票** — 所有其他发票（Filing Fee、差旅、办公等）

### 去向规则

- 律所发票 → `00-已确认/00-律所发票/`（识别后直接归入）
- 个人发票 → `00-已确认/00-个人发票/`（识别后直接归入）
- USPS跳过 → `00-已确认/00-USPS跳过/`
- **普通发票 → 进入待复核流程**

### 铁律

- **所有文件都要经过识别和判断去向，不能跳过**
- **普通发票不能跳过待复核，必须经过人工确认**
- 个人发票和律所发票虽然不分组，但也需要识别信息并登记到Excel

## 五、待复核（仅普通发票）

### 分组规则
- 每20个一组，分成 `第01组`、`第02组`...（两位前导零，macOS排序正确）
- 每组文件夹里放20个PDF原件
- 优先按类别排序，同类放一起方便批量审核

### Excel登记
- 生成Excel登记记录，包含：序号、日期、分类、收款方、付款方、金额、文件名、状态、备注
- Excel保存到项目根目录：`00-发票总索引-YYYYMMDD.xlsx`

## 六、复核通过

人工复核确认无误后：
- 普通发票 → 文件归入 `00-已确认/` 下，Excel状态更新为"已确认"
- 文件名字段不对的（收款方付款方互换等）→ 改名+改Excel
- 律所发票 → 文件归入 `00-已确认/00-律所发票/`
- 个人发票 → 文件归入 `00-已确认/00-个人发票/`

## 七、归档

全部通过后：
- Excel全部状态为"已确认"
- 00-待处理和 00-待复核清空
- Excel最终状态与 00-已确认 文件夹一一匹配，记录数等于文件夹文件数

## 八、Excel登记

### 表格结构（9列）

| 列 | 字段名 | 说明 |
|---|---|---|
| 1 | 序号 | 从1开始连续编号 |
| 2 | 状态 | 待处理 / 待复核-第NN组 / 已确认 |
| 3 | 凭证文件名 | 完整文件名 |
| 4 | 日期时间 | 12位数字（YYYYMMDDHHMM） |
| 5 | 费用类型 | 如 Filing Fee、Travel Expense |
| 6 | 收款方 | 发票开具方 |
| 7 | 付款方 | 实际付款人 |
| 8 | 金额 | 纯数字 |
| 9 | 备注 | 可选，异常情况说明 |

### 格式规范
- 表头：微软雅黑11号加粗白色，背景蓝色(#4472C4)，居中对齐
- 数据行：微软雅黑10号细线边框
- 状态列：绿色背景(#C6EFCE)深绿色文字(#006100)居中
- 金额列：右对齐格式#,##0.00
- 排序：按日期时间升序排列

### 写入后必须验证
- 用 load_workbook 重新读取验证行数、金额、批次是否正确
- 常见错误：1）写入时索引偏移导致漏写一行；2）重复写入导致总计行变成数据行被求和
- 验证方法：逐行求和 + 对比总计行 + 检查批次计数是否匹配

## 十、坑点记录

### Wave Financial识别
- 原始 PDF 收款方：Wave Financial Inc.
- Stripe 商家名：Wave Team
- 正确命名：Wave Financial Inc.（不是 The Wave Team）
- 分类：Computer-Software

### LSCO 分类
- AI 自动分类经常出错，$98.8 应该是 Shipping-Fee
- 必须人工判断

### Filing ID 反查问题
- 程序通过 Filing ID 查询州务卿数据库反查公司名
- 反查出的公司名**不放入文件名**
- 例如：Acous LLC 是通过 Filing ID 反查的，原始 PDF 没有这个名称

### 政府收据
- 政府收据（MA State, NY State, DE State）：
  - 收款方 = 政府机构（如 Commonwealth of Massachusetts）
  - 付款方 = 实际付款人（如 Xiaoxiao Liu）
  - Filing ID 反查的公司名不放入文件名

### 律所发票
- WUKONG 等律所自开发票不参与客户发票识别流程
- 单独存放在 `00-已确认/00-律所发票/` 文件夹

### 重复文件
- 同一 Invoice 号的不同命名文件视为重复
- 保留一份，删除其余

### 金额提取
- 注意全角$（＄）和带空格的情况
- 金额可能是 NTD 或其他货币，不是所有都是 USD

## 十一、批次管理规范

### 文件夹命名
- 新批次目录命名：`YYYYMMDD-名称-v版本`（如 `20260611-Receipts-v1`）
- 每次新批次只移动 `00-待处理` 和 `00-待复核`，不动 `00-已确认`

### 分组命名
- 两位前导零：`第01组`、`第02组`...`第82组`
- 原因：macOS Finder按字母排序，不加前导零会把第10组排在第2组前面

## 十二、API 调用常见问题

### Vision API 报错
- **401 Unauthorized**：API Key 不正确或缺失
- **429 Too Many Requests**：请求太快被限流，增加延迟
- **Timeout**：图片太大或网络问题，检查图片大小（建议 < 5MB）
- **识别不准确**：降低 temperature 到 0.05，或提高图片分辨率

### PDF 处理
- 有文本层的 PDF → 用 PyMuPDF 提取文本，不花钱
- 扫描件 PDF → 先转图片（fitz.Matrix 2.0x），再调 Vision API
- 图片格式（jpg/png）→ 直接调 Vision API
