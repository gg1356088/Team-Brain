# 系统支撑概述

## 一句话总结

这一层不是业务，但支撑所有业务的运行。包括 AI Agent 规则、API 凭据管理、本地工具和技能索引。

## 它管什么

### 1. 🤖 Agent 规则

告诉 AI（Claude Code、Codex）在这个工作区里应该怎么做事。

- **AGENTS.md**：顶层路由规则和安全边界，最高优先级
- 历史行为规则仅供参考，当前规则以 AGENTS.md 为准

### 2. 🔑 API 凭据和基础设施

管理 Google API、邮件、Gemini 等外部服务的连接配置。

- **api_registry.py**：集中式 API 认证模块，12 个 API 的统一入口
- Google OAuth 桌面应用，Shared Drive 结构说明
- Obsidian MCP 工具配置
- 安全规则：可以记录凭据文件名，但绝不记录 secret/token 内容

### 3. 🛠️ 本地工具

| 工具 | 用途 |
|------|------|
| Offline Workflow Dashboard | 本地 Web 面板，展示所有 workflow 状态 |
| Code Graph Builder | 生成代码结构图，帮助 Agent 快速路由 |
| Open Office Web | Office 文件用浏览器优先打开 |
| Self-run Inventory | 列出所有自运行程序清单 |

### 4. 📋 技能索引

记录本系统具备的全部能力，回答"这个系统能做什么"时用。

## 关键原则

- 系统支撑不是第四类业务
- 普通任务先选人事/行政/财务入口
- 只有路由、排错、工具维护时才进这层
- Google 生产操作默认走 Admin API（HR workflow 例外）
