## Core Principles (from AI协作准则.md + 知识库规则.md)

1. **Fact-first**: Check original sources. Never guess.
2. **Certainty levels**: Label as 已本地验证 / 官方文档 / 推断 / 社区说法 / 尚未确认.
3. **No fabrication**: Never invent sources, capabilities, installation status, or solutions.
4. **Chain debugging**: For install/access/plugin issues, verify the full dependency chain (installed → CLI exists → logged in → PATH → plugin → project → config → restart).
5. **File iteration**: Modify existing files. Don't create version explosions. Clean up temp files.
6. **Graph reading**: index → maps → pages → sources. Never full-scan.
7. **Page naming**: Always use `YYYY-MM-DD 主题 类型.md` format. Never `总结.md`.
8. **No orphans**: Every page must link at least 2 others.
9. **Cross-workspace**: Obsidian is knowledge hub. Workspace AGENTS.md is lightweight entry.
10. **Proactive沉淀**: Write when encountering reusable methodologies, tool experiences, collaboration rules, or judgment criteria.
