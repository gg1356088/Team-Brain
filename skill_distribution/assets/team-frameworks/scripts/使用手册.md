# 发票自动化工作流 — 使用手册

> 版本：v1.0 | 更新日期：2026-06-15
> 适用对象：需要操作发票PDF/图片分类整理的任何人

---

## 一、这是什么

这是一套发票整理工作流程。把收到的各种发票（PDF、图片等）按类型分类、提取关键信息、人工复核、最终归档。

**核心目标：** 让1500+张发票有序可查，每笔费用都能追溯到。

---

## 二、整体流程

```
│  00-待处理  │───→│  识别信息 │───→│ 判断去向  │───→│  归入目标   │
│ (原始文件)  │    │          │    │          │    │  文件夹     │
└─────────────┘    └──────────┘    └──────────┘    └─────────────┘
                                                        │
                     ┌──────────────────────────────────┤
                     ▼                                  ▼
            ┌──────────────┐                   ┌──────────────┐
            │ 00-待复核分组 │──人工确认──→     │ 00-已确认    │
            │ (每20个一组) │                   │ (最终归档)   │
            └──────────────┘                   └──────────────┘
                                                │
                            ┌───────────────────┼───────────────────┐
                            ▼                   ▼                   ▼
                   00-律所发票/         00-个人发票/         00-USPS跳过/
```

### 五种文件去向

| 去向 | 说明 | 典型文件 |
|---|---|---|
| **00-律所发票** | 律所自开发票，不参与客户流程 | WUKONG律师费发票 |
| **00-个人发票** | 个人消费，非公司业务 | currency exchange、cash withdraw |
| **00-USPS跳过** | 无法处理或特殊标记 | USPS信封照片 |
| **00-待复核** | 普通发票，需人工确认 | Filing Fee、Gas、Meal等 |
| **00-已确认** | 人工确认后最终归档 | 所有通过的文件 |

## 四、操作步骤

### 步骤1：处理待处理文件夹

待处理文件夹里有按费用类型分的子目录，每个子目录下有原始文件（PDF/jpg/png）。

**操作：**
1. 打开 `00-待处理/` 下的子目录
2. 对每个文件进行识别：
   - **PDF且有完整文件名** → 从文件名提取信息（秒级）
   - **缺字段PDF或图片** → 用Vision工具识别内容

### 步骤2：判断文件去向

识别后根据费用类型判断归入哪个文件夹：

**直接归入（不需复核）：**
- 律所发票 → `00-已确认/00-律所发票/`
- 个人发票 → `00-已确认/00-个人发票/`
- USPS跳过 → `00-已确认/00-USPS跳过/`

**进入待复核：**
- 普通发票（Filing Fee、Gas、Meal等）→ 进入待复核分组

### 步骤3：待复核分组

普通发票按20个一组分组：

**分组规则：**
- 每组20个PDF文件
- 组名用两位前导零：`第01组`、`第02组`...`第82组`
- 每组文件夹里放20个PDF原件
- 优先按类别排序，同类放一起

**为什么要两位前导零？**
macOS Finder按字母排序。不用前导零的话，`第10组`会排在`第2组`前面，`第82组`会排在`第10组`前面，完全乱序。

### 步骤4：人工复核

这是最关键的一步——**人工确认每张发票的信息是否正确**。

**复核内容：**
1. 分类是否正确（Filing Fee、Gas、Meal等）
2. 收款方/付款方是否颠倒
3. 金额是否匹配
4. 日期是否准确

**复核通过后的操作：**
- 普通发票 → 文件移入 `00-已确认/`
- 律所发票 → 文件移入 `00-已确认/00-律所发票/`
- 个人发票 → 文件移入 `00-已确认/00-个人发票/`
- 发现问题 → 修正文件名后重新放回待复核

**铁律：不能跳过这一步！** 已确认文件夹里的文件必须经人工复核后才能移入。

### 步骤5：Excel登记

每处理一个文件，都必须在总索引表中登记。

**登记字段（9列）：**

| 序号 | 状态 | 凭证文件名 | 日期 | 费用类型 | 收款方 | 付款方 | 金额 | 备注 |
|---|---|---|---|---|---|---|---|---|
| 1 | 已确认 | 202508210053-Filing Fee-DE DCIS-IncepVision-$100.pdf | 20250821 | Filing Fee | DE DCIS | IncepVision | 100 | |
| 2 | 待复核-第01组 | ... | ... | ... | ... | ... | ... | ... |

**状态字段取值：**
- `待处理` — 还没处理的文件
- `待复核-第NN组` — 在待复核分组中的文件
- `已确认` — 人工复核通过的文件

## 五、命名规则

### 标准文件名格式

```
YYYYMMDDHHMM-分类-收款方-付款方-$金额.扩展名
```

### 字段说明

| 字段 | 格式 | 示例 |
|---|---|---|
| 日期时间 | 12位 YYYYMMDDHHMM | `202508210053` |
| 分类 | 标准分类名 | `Filing Fee`、`Tax Payment` |
| 收款方 | 开票机构全称 | `Wave Financial Inc.` |
| 付款方 | 实际付款人 | `IncepVision` |
| 金额 | $开头 | `$19`、`$1,020.04` |
| 扩展名 | 原文件类型 | `.pdf`、`.jpg`、`.png` |

### 示例

```
202508210053-Filing Fee-DE DCIS-IncepVision-$100.pdf
202408041619-Gas-崇德盈加油站-General-NTD584.jpg
202605280414-Computer-Software-Wave Financial Inc.-IncepVision-$19.pdf
```

## 六、常见问题

### Q1：个人发票和律所发票需要复核吗？

**答：不需要分组复核，但需要识别信息并登记Excel。**

律所发票和个人发票直接从待处理识别后归入对应文件夹，但也要在Excel中登记。

### Q2：为什么文件都在待处理文件夹的子目录里？

**答：** 这是之前按费用类型自动分类的结果。每个子目录是一种费用类型（如`gas/`、`travel/`、`meal and entertainment/`）。实际处理时需要遍历所有子目录。

### Q3：待复核有多少组？

**答：** 当前有82组，每组20个PDF（最后一组3个）。从第01组开始依次复核。

### Q4：遇到看不清的图片怎么办？

**答：** 用Vision工具识别。如果识别结果仍不明确，标记为"需人工确认"，在Excel备注中说明。

### Q5：金额提取要注意什么？

**答：**
- 全角$（＄）要替换为半角$
- 金额可能有空格：`- $300` → `$300`
- CAD等其他货币要单独标记
- 逗号要去掉：`$1,020.04` → `1020.04`
- 文件名末尾的`(1)`是多页标记，不是金额一部分

## 七、注意事项

1. **不要跳过待复核** — 普通发票必须经人工确认后才能归入已确认
2. **分组命名用两位前导零** — `第01组`不是`第1组`
3. **Excel必须同步更新** — 文件移入已确认后，Excel状态也要更新
4. **不要误删文件** — 操作前确认文件还在，别凭记忆下结论
5. **遇到不确定的先问** — 分类拿不准、金额有疑问，标记后人工判断
