# 发票自动化 — 团队使用指南

> 先读框架理解方法论，再用脚本干活。脚本本身不依赖 Hermes，任何人装上 Python 就能跑。

## 一、准备环境

```bash
pip install pymupdf openpyxl requests
```

## 二、脚本清单

| 脚本 | 位置 | 作用 | 依赖 |
|------|------|------|------|
| `invoice_recognize.py` | `scripts/` | 独立识别脚本，改 API_KEY + PROJECT_DIR 就能跑 | Agnes Vision API |
| `invoice_workflow.py` | `scripts/` | 完整五步流水线：audit→parse→classify→group→report | 同上 |

## 三、五步流水线

```bash
cd /你的项目目录

# 1. 审计当前状态
python3 invoice_workflow.py audit

# 2. 解析文件名提取信息
python3 invoice_workflow.py parse

# 3. 按规则分流
python3 invoice_workflow.py classify

# 4. 待复核分组（每20个一组）
python3 invoice_workflow.py group

# 5. 生成报告
python3 invoice_workflow.py report
```

## 四、配置方式

脚本通过环境变量读取配置，不硬编码路径：

```bash
# Agnes Vision API（用于扫描件识别）
export AGNES_API_KEY="你的key"
export AGNES_API_URL="https://apihub.agnes-ai.com/v1/chat/completions"
export AGNES_MODEL="agnes-2.0-flash"

# 项目目录（脚本的工作目录）
export INVOICE_PROJECT_DIR="/你的项目路径"
```

## 六、典型工作流

```
第一天：
  1. 读框架 → 2. 配环境变量 → 3. python invoice_workflow.py audit → 看清现状

第二天起（日常）：
  1. python invoice_workflow.py parse → 从文件名提取信息
  2. python invoice_workflow.py classify → 按规则分流
  3. python invoice_workflow.py group → 分组到待复核
  4. 人工审核每个分组
  5. 审核通过后移入已确认
  6. python invoice_workflow.py report → 生成总览
```

## 七、重要提醒

- 脚本里的费用类型别名映射已内置 40+ 种变体，不需要手动维护
- 分类规则直接看脚本 docstring 和 ALIAS_MAP 字典
- 遇到新坑 → 先记到 vault 对应笔记 → 再更新团队框架 → 最后更新脚本 ALIAS_MAP
