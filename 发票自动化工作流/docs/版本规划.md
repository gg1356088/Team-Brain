# 发票自动化工作流 — 版本规划

## V1：正则解析文件名（✅ 已完成）
- **能力**：从文件名正则提取日期、金额、分类信息
- **脚本**：`发票自动化工作流/脚本/invoice_workflow.py`（V2.0）
- **输入**：已命名的发票文件
- **输出**：结构化 JSON + Excel 登记
- **局限**：依赖文件名规范，乱名就解析失败

## V2：Vision 复核 + PDF 内容提取（🔧 进行中）
- **能力**：
  - Vision API 复核正则解析结果，纠正错误
  - PDF 内容提取（PyMuPDF + OCR fallback）
  - 图片发票识别（vision_analyze）
- **脚本**：`发票自动化工作流/invoice_recognize.py`（V2.0）
- **输入**：原始发票 PDF/图片
- **输出**：结构化 JSON + 复核标记
- **关联**：`fix_invoice_v3.py`、`check_invoice_content.py` 属于 V2 的修复/调试脚本

## V3：邮件自动拉取 + Google Drive 上传（📋 待开发）
- **能力**：
  - 从 IncepVision Office 邮箱自动拉取发票附件
  - 自动上传到 Google Drive 共享盘
  - 端到端自动化：邮件 → 下载 → 识别 → 分类 → 上传
- **脚本**：待新建，参考 `supervisor/invoice_workflow_template.py`
- **前置依赖**：GWS 认证（`setup_gws_auth.py` 已存在）

## V4：全自动流水线（🎯 远景）
- **能力**：
  - 定时任务自动运行（cron job）
  - 多邮箱源聚合（IncepVision + 其他）
  - 异常告警 + 人工干预入口
  - 版本迭代管理 + 数据回滚
- **触发**：V3 稳定运行后启动
