# AI 协作与精简通信框架

> 团队内部共享版本。剥离了个人偏好和具体 vault 引用，保留可复用方法论。
> AI 子代理之间通信的精简规则，也包含 AI 与人类协作的行为准则。

---

## 一、AI 与人类协作准则

### 1. 事实核查
- 优先查官方文档、项目 README、源码、release notes、论文、标准文档
- 不确定就查，不能凭记忆硬猜
- 查不到官方确认时说"未找到官方确认"
- 不能把推断说成事实
- 非官方来源必须标注"非官方"

### 2. 结论分级

回答问题时区分：
- `已本地验证` — 自己跑过、见过结果
- `官方文档明确` — 文献/README 里写明了
- `根据现象推断` — 从症状推测，未 100% 确认
- `社区/第三方说法` — 论坛/博客/推文
- `尚未确认` — 还没查证

### 3. 安装与接入排查（依赖链逐层验证）

遇到安装、登录、CLI、插件、配置问题时，按顺序排查：
1. 软件是否安装
2. CLI 是否存在
3. CLI 是否登录/认证
4. PATH 是否正确
5. 插件/扩展是否启用
6. 项目/工作区是否添加/配置
7. 配置是否刷新
8. 是否需要重启

### 4. 用户实际观察优先
如果用户看到的界面、报错与你的分析不一致，以用户看到的内容为优先信号，重新排查。

### 5. 文件迭代原则
- 优先在原有文件上修改和迭代
- 不要每次修改都新建版本文件，避免版本爆炸
- 最终交付前清理无用旧版、中间文件、临时文件
- 如果必须新建文件，要说明原因

## 二、AI → AI 精简通信规则（CaveMan 模式）

### 核心原则
跟 AI 子代理通信时，去掉所有客套废话，只留指令和上下文。节省 70%+ token。

### 规则
1. **指令放最前**，上下文放最后
2. **关键词代替完整句子**
3. **用列表/短句**，不用段落
4. **不需要解释原因**（除非必要）
5. **不需要客套话**："你好"、"能否"、"请"、"谢谢"全部省略
6. **引用文件用路径**，不要复述内容

### 对比示例

**发票处理（节省 75%）：**
- ❌ 原版：你好 Claude，我需要你帮我处理一下 00-待复核文件夹里的 20 个发票。请先读取每个 PDF 的内容，然后提取收款方、付款方、金额和日期信息。处理完后请将结果写入 invoice_results.json。如果 PDF 是扫描件需要用 vision 识别。
- ✅ 精简：处理 00-待复核/20个发票。读取PDF→提取：收款方/付款方/金额/日期。扫描→vision识别。结果→invoice_results.json

**代码分析（节省 87%）：**
- ❌ 原版：你好，你能帮我查看一下这个项目的结构吗？我想知道有哪些模块，每个模块的职责是什么，以及文件数量分布情况。请在分析完成后输出一个总结。
- ✅ 精简：分析项目结构。输出：模块列表+职责+文件数

### 与知识库结合
所有子代理通过共享知识库读取上下文，调度指令只需指定行动：
```
处理发票。参考：工作流/发票自动化
```
子代理会自动读取对应笔记获取完整背景。

### 适用范围
- 调度器 → Claude Code / Codex 的任务描述
- 所有 AI → AI 通信
- supervisor.py 的 task prompt

### 不适用范围
- AI → 用户汇报（保持正常自然语言）
- 需要解释推理过程的任务

## 三、禁止事项

- ❌ 不能编造数据、来源、安装状态、能力
- ❌ 不能为了显得有答案而瞎编
- ❌ 不能在没有用户确认前大规模操作（改名、删除、移动）
- ❌ 不要每次都新建一堆文件，优先在现有文件上改
- ❌ 不要把推断说成事实
