# 团队框架包

> 从个人 Obsidian 知识库沉淀中剥离出的可复用框架。去除了所有具体公司名/金额/路径/邮箱，保留方法论、规则和踩坑经验。
> 新成员接手项目时先读这些，避免重复踩坑。

## 使用顺序

1. 先读 `团队Hermes接入指南.md` — 装上 Hermes，加载技能
2. 再读 `框架与Vault对照.md` — 了解每个框架从哪来
3. 然后读具体框架 — 建立方法论认知
4. 最后去 vault 看原文 — 看具体案例

看具体案例 → 对照此表去 vault 找原始笔记
看方法论 → 直接读框架

## 与外部分发包的区别

| | 外部企业版 (`skills-obsidian/`) | 团队框架版 (`team-frameworks/`) |
|---|---|---|
| 受众 | 其他公司、开源社区 | 内部团队成员 |
| 内容 | Hermes 技能文件（去硬编码） | 方法论 + 踩坑经验 + 规则 |
| 数据剥离 | 只去硬编码路径 | 去掉硬编码个人路径/金额/API key（公司名保留，因为是自己团队的） |
| 形式 | 可安装的技能包 | Markdown 文档，直接读 |

## 使用方式

1. 新成员入职 → 先读完这 6 个框架
2. 接手具体项目 → 再读对应的详细文档（Obsidian vault 或项目 README）
3. 遇到新坑 → 补充到对应框架的坑点部分
4. 变更流程 → 更新框架后同步给所有人
