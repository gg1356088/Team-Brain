# 人事模块概述

## 一句话总结

这个模块负责招聘全流程自动化：从接收简历邮件，到提取信息填入 Google Sheets，再到安排面试、管理笔试队列，最后通过 Google Chat 汇报状态。

## 它能做什么

| 功能 | 怎么做到的 |
|------|-----------|
| 📧 简历抓取 | 自动扫描新浪邮箱和 Gmail 中的应聘邮件，下载简历附件 |
| 👤 信息提取 | AI（Gemini）读取简历 PDF，提取姓名、学历、工作年限等信息 |
| 📊 写入表格 | 自动填入 Google Sheets "1.1.7 应聘者信息表" |
| 📝 面试邀约 | 通过 Gmail API 发送面试邮件，含 Zoom 链接和问卷 |
| 🧠 笔试管理 | 发送笔试题、收集答卷、记录结果 |
| 📢 状态汇报 | 通过 Google Chat webhook 实时汇报招聘进度 |
| 🔄 自动复检 | macOS LaunchAgent 定时运行，检查是否有遗漏或卡住的流程 |

## 关联的 Google 服务

- **Sheets**：1.1.7 应聘者信息表（主数据表）
- **Gmail**：发送面试邀请（team@incepvisionlaw.com）
- **Drive**：存储简历 PDF
- **Chat**：状态汇报和告警
- **Docs**：H71 面试官配置表、H73 面试预约 Ledger

## 两个招聘渠道

| 渠道 | 岗位 | 说明 |
|------|------|------|
| Gmail | 法律实习生 | 主渠道，流程最成熟 |
| 新浪邮箱 | 行政助理 | 独立链路，A2 生效中（A1/A3/A4 已废止） |

## 日常操作

- 正常运行时无需手动干预，LaunchAgent 自动处理
- 如需查看状态：打开 Sheets 或看 Chat 汇报
- 遇到问题时：先查运行日志，不要重新设计流程

## 技术备注

- 这是一个成熟的 Python self-running workflow
- 代码入口：`workflows/hr-resume-automation/`
- 运行在 Mac Mini 上
- 使用 HR/Team 专用 Google 身份（非 Admin 身份）
